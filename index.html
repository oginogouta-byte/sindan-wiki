System Flow

システムフロー

診断完了からレポートが届くまでの流れ。CVトリガーはUTAGE（500円）またはX（引用RT）の2案。

① 診断HTML（100問回答）
ユーザーが100問に回答。完了時に診断ID + 回答データをGoogle Apps ScriptへPOST。
② Google Sheets（データ保存）
GASがスプレッドシートに1行書き込み。診断IDをキーとして管理。
③ UTAGEサンクスページ（共通起点）
診断完了後に遷移。2つのCVトリガーを提示する分岐点。診断IDをパラメータで引き継ぐ（技術検証中）。
④ CVトリガー分岐（2案）
案A（UTAGEドメイン）：500円決済フォーム → 決済完了 → メール・氏名取得 → stateで診断ID引き継ぎ。
案B（Xドメイン）：固定ツイートのリンク → 引用RT → 診断IDと照合 → XStep DM送信。
⑤ GAS：診断ID × ユーザーID 紐づけ
いずれのルートでも診断IDとユーザー情報をGASで照合・Sheetsに記録。
⑥ Claude API：レポート生成
3層変数（Layer1〜3）をClaude APIに渡してセクション別テキストを生成。Sheetsに保存 → 個別URL発行。
⑦ レポート送付 → LINE登録CTA
案A：メールで届く / 案B：XStep DMで届く。レポート末尾でLINE公式アカウントへ誘導。
ポイント：AI生成は閲覧時ではなくデータ受信時（1回のみ）に実行。コスト固定・表示速度を確保。
Scoring

スコアリング設計

各観点25問 × 最大5点 = 各観点125点。4観点の内部合計（最大500点）を非線形変換し、AI戦闘力として表示する。

戦闘力の算出式（軸別加算型）：
各軸トークン = round( (軸スコア ÷ 125)³ × 249,999 ) AI戦闘力（総合）= 知識力 + 思考力 + 活用力 + 判断力（各軸トークンの合計）

4軸それぞれで3乗変換し合算。各軸最大 249,999トークン、総合最大 999,999トークン。バックエンドは生スコア（1〜5）保持、表示変換はフロント側で実施。

ランク AI戦闘力（表示値） 内部スコア換算 レベル感
S 729,000 〜 999,999トークン 約450点以上 AIを完全に使いこなしている最上位層
A 343,000 〜 728,999トークン 約350〜449点 業務でAIを高度に活用できている
B 125,000 〜 342,999トークン 約250〜349点 ある程度使えているが伸びしろあり
C 27,000 〜 124,999トークン 約150〜249点 使い始めている段階。基礎の強化が必要
D 〜 26,999トークン 約149点以下 AI活用がほぼできていない。ここからがスタート
※ランク判定は内部スコア（500点満点）ベースで行い、ユーザーへの表示は戦闘力に変換する。境界値は実際のスコア分布データを取得後に調整予定。

Report Design

レポート全体構成

縦スクロール型・個別レポート。性格分類ではなく「市場内の能力位置づけ」をパフォーマンス分析として提示する。

設計思想：タイプ名・ランクなし。「市場の中で自分がどこにいるか」を上位X%で即時把握させ、サブカテゴリの数値と言語化の組み合わせで「このレポートは自分のことを全部分かっている」という体験を作る。デザイン品質がコンテンツ品質に見える。診断ページと同水準のビジュアルで、業界の基準を作りにいく。
コンバージョン設計：レポート体験の質そのものがLINE登録のドライバー。低スコア層は「情報が欲しい」、高スコア層は「成長意欲が刺激される」という2つのパスで自然に誘導する。CTAは「1on1誘導」ではなく「個別最適化された情報をLINEでお届け」。

Evaluation Structure

評価の3層構造

個別最適化レポートを生み出すための3つの評価レイヤー。MBTIのような事前テキストマッピングではなく、Claude APIが3層の文脈から都度生成する。

根本方程式：アウトプット = 思考フレームの質 × AI実行能力
AIは「増幅装置」であって「補完装置」ではない。弱い思考を強くはしないが、強い思考を大量に実行してくれる。
AIが提供する3つの価値
① 速度と量— 自分の思考を分身として再現・量産する（100人でやっていたことを1人で）
② スキル実行— 自分にないスキルセットを実行できる（デザイン・コード・法務…）
③ 思考の外部化— 自分の思考フレームをAIに設計・再現させる
Layer 1 — 数値
スコア（軸スコア + サブカテゴリ内訳）
各観点スコア（/125）をサブカテゴリに分解することで「知識力が高い」ではなく「ツール知識は高いが本質理解が低い」という粒度を得る。サブスコアの組み合わせパターンがClaude APIへの入力文脈になる。
例：知識力 88/125 → ツール知識（高）/ 本質・限界理解（低）/ 動向・発展把握（中）
Layer 2 — パターン
ユーザー分類タイプ（サブカテゴリのパターンから判定）
軸スコアの合計ではなく、サブカテゴリの組み合わせパターンから分類タイプを付与。「思考設計（高）× 時短実感（低）」と「時短実感（高）× 思考設計（低）」は同じ合計でも異なるタイプとして評価する。分類アルゴリズムは別途設計中。
例：思考設計（高）× スキル実行（高）× 量産統合（高）→ 上級活用型
Layer 3 — 矛盾
整合性チェック（自己認識と実態のズレ）
9グループのトラップ問により「言っていることとやっていることの不一致」を検出。思考フレームの質が問われる核心部分。ここが最も「刺さる」個別要素。
例：判断-1グループ不整合 → 「AIの出力確認に対する判断基準がまだ自分の中で定まっていないサインです」
